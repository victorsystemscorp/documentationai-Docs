## Descripción general

<Callout kind="alert" collapsed="false">
  Este sitio incluye contenido de ejemplo. Usa este inicio rápido para reemplazarlo con la documentación real de tu producto.
</Callout>

## 1. Actualiza una página

<Steps>
  <Step title="Editar introducción" icon="edit" title-type="p">
    Reemplaza la introducción de ejemplo con una descripción concisa de tu producto y las acciones clave que quieres que los nuevos usuarios realicen primero.
  </Step>

  <Step title="Previsualizar tu cambio" icon="monitor" title-type="p">
    Usa el editor del panel de control para confirmar que la introducción actualizada se ve y se lee como esperas.
  </Step>
</Steps>

## 2. Agrega una nueva página

<Steps>
  <Step title="Agregar una página" icon="file-text" title-type="p">
    Crea una nueva `page` haciendo clic en el botón + de la barra de navegación izquierda del panel. Si usas git, crea el archivo .mdx y añádelo a la barra de navegación usando documentation.json.
  </Step>
</Steps>

## Flujo de incorporación de ejemplo (reemplázalo con el tuyo)

1. Regístrate para obtener una cuenta y verifica tu correo electrónico.

2. Crea tu primer espacio de trabajo, proyecto o entorno.

3. Agrega una integración o fuente de datos para conectar tus herramientas existentes.

4. Completa una acción guiada (por ejemplo, ejecuta tu primer trabajo o publica tu primer recurso).

5. Invita a un integrante del equipo y asígnale un rol o permiso básico.

## 3. Conecta la referencia de API (opcional)

Si tienes una API, reemplaza `openapi.yaml` con tu archivo OpenAPI real. Si no necesitas una referencia de API, elimina esa pestaña desde el panel o desde `documentation.json`.
