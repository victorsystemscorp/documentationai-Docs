## Descripción general

Este es contenido de muestra. Reemplaza los tokens, las URL, los encabezados y los flujos con los requisitos reales de autenticación de tu API.

Esta API usa OpenAPI para describir endpoints, parámetros y respuestas. La referencia se genera a partir de `openapi.yaml`, así que mantén ese archivo actualizado a medida que tu API evoluciona.

## Autenticación

Usa un token bearer en el encabezado `Authorization` para las solicitudes autenticadas. Reemplaza la URL base y el token con los valores de tu propia API.

```bash
curl https://api.example.com/v1/example \
  -H "Authorization: Bearer YOUR_TOKEN"
```

## Mantener OpenAPI sincronizado

Define tu autenticación y la configuración base en `openapi.yaml` para que la referencia de la API se mantenga precisa y coherente.

Como mínimo, deberías:

- Definir uno o más esquemas de seguridad que describan cómo se autentican los clientes (por ejemplo, token bearer, clave de API en un encabezado u otro esquema).
- Hacer referencia a esos esquemas de seguridad en las operaciones que requieren autenticación para que quede claro cuáles están protegidas.
- Establecer la URL base correcta y cualquier encabezado global obligatorio (por ejemplo, encabezados de versionado o de inquilino) para que los ejemplos de solicitud se puedan copiar y pegar sin cambios.

Cuando cambie tu flujo real de autenticación (por ejemplo, nuevos nombres de encabezado, formatos de token o requisitos adicionales), actualiza `openapi.yaml`. La referencia de la API generada reflejará entonces tu modelo de autenticación actual sin tener que editar manualmente cada endpoint.

## Uso de la referencia de API

Usa la referencia de la API para explorar endpoints, parámetros obligatorios y respuestas de ejemplo. Cuando cambies los encabezados de autenticación o las URL base, actualiza `openapi.yaml` para que la referencia generada coincida con lo que tus clientes deben utilizar.

## Errores comunes

<Expandable title="¿Por qué estoy recibiendo un 401?" default-open="false">

Verifica que:

- El token sea válido y no haya expirado.
- El token se envíe en el encabezado `Authorization`.
- El valor del encabezado incluya el prefijo `Bearer` y un espacio antes del token.

</Expandable>
