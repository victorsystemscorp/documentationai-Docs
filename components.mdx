### Callout: consejos de uso

**Cuándo usarlo**

- Resaltar advertencias o restricciones importantes que afectan el éxito
- Señalar consejos que ahorran tiempo o evitan errores frecuentes
- Enfatizar prerrequisitos, permisos o acciones irreversibles

**Cuándo no usarlo**

- Para cada detalle menor; el abuso hace que las advertencias reales se ignoren
- Como sustituto de una redacción clara y buenos encabezados
- Para ocultar información crítica y bloqueante al final de una página larga

### Steps: consejos de uso

**Cuándo usarlo**

- Cuando las personas deben seguir acciones en un orden específico
- Cuando describes tareas que se completan en una sola sesión
- Cuando quieres separar la explicación conceptual de las acciones prácticas

**Cuándo no usarlo**

- Para listas de verificación donde el orden no importa
- Para documentar conceptos de fondo o arquitectura
- Para flujos largos y ramificados que necesitan su propia guía dedicada

### Tabs: consejos de uso

**Cuándo usarlo**

- Cuando muestras la misma idea en varios lenguajes o plataformas
- Cuando ofreces rutas de configuración alternativas, como CLI frente a UI
- Cuando quieres mantener variantes breves en una sección compacta

**Cuándo no usarlo**

- Cuando una variante es claramente más recomendable que las demás
- Para flujos largos e independientes que merecen páginas separadas
- Cuando el contenido difiere tanto que las personas deben leer todas las variantes de todos modos
